## About

Our emojis are using the same naming convention as Slack, the data is taken from [emoji-data], we use Google & Apple variants.

They're used internally in our [EmojiPicker component](https://www.welcome-ui.com/components/emoji-picker)

## Default

`<Emoji />` takes an `emoji` prop that is the name of the emoji following Slack & [emoji-data] naming convention, it can be wrapped with colon or not. It can also take a url instead, it can be usefull when creating custom `<EmojiPicker.List />`.

```jsx
<Stack direction="horizontal">
  <Emoji emoji=":smile:" />
  <Emoji emoji="dog" />
  <Emoji emoji="https://emoji.slack-edge.com/TSTNFTZAP/slack/7d462d2443.png" />
</Stack>
```

## Size

`<Emoji />` are images, they can take `width` & `height` props or `size` which is a shortcut for both.

```jsx
<Stack direction="horizontal" alignItems="center">
  <Emoji emoji=":child:" size={12} />
  <Emoji emoji=":woman:" size={24} />
  <Emoji emoji=":older_woman:" size={36} />
</Stack>
```

## Theming & SSR

We rely on [bowser](https://www.npmjs.com/package/bowser/v/2.11.0) to check the user's OS, if it's an Apple one then we will display Apple's emojis, otherwise Google's (like Slack). You can override this with the `useAppleEmoji` prop, it can be useful to force Google or Apple emojis and is needed for SSR.

```jsx
<Stack direction="horizontal">
  <Emoji emoji=":apple:" useAppleEmoji />
  <Emoji emoji=":bug:" useAppleEmoji={false} />
</Stack>
```
